# subscription-handler

Telegram-бот для управления подписками на объявления о квартирах. Принимает команду `/subscript`, проводит пользователя через серию вопросов и сохраняет фильтры в таблице `user_subscriptions`. При повторной подписке старая запись деактивируется и создаётся новая.

## Команды бота

| Команда | Действие |
|---|---|
| `/subscript` | Запустить мастер создания подписки |
| `/cancel` | Прервать текущий диалог |

## Диалог создания подписки

После `/subscript` бот последовательно задаёт 6 вопросов (с `ForceReply` — поле ввода фокусируется автоматически). На каждый шаг нужно ввести число; **0 означает «без ограничения»**.

```
Шаг 1/6 — Минимальная цена аренды (₽)?
Шаг 2/6 — Максимальная цена аренды (₽)?
Шаг 3/6 — Минимальная площадь (м²)?
Шаг 4/6 — Максимальная площадь (м²)?
Шаг 5/6 — Количество комнат?   ← числа через пробел или запятую, 0 = любое
Шаг 6/6 — Минимальный скор квартиры?
```

После последнего ответа бот сохраняет подписку и присылает сводку:

```
✅ Подписка успешно создана!

💰 Цена: 50000 — 85000 ₽
📐 Площадь: 38 — любая м²
🚪 Комнат: 2, 3
⭐ Мин. скор: 0
```

Если при вводе допущена ошибка (не число, некорректное значение), бот просит ввести ответ повторно — шаг не продвигается.

## Как работает сохранение

1. Деактивируется любая существующая активная подписка для данного `chat_id` (`is_active = FALSE`)
2. Вставляется новая строка с `is_active = TRUE`

Обе операции выполняются в одной транзакции.

## Конфигурация

```yaml
telegram:
  bot_token: ""   # токен бота от @BotFather

database:
  dsn: "postgres://realty_parser:password@realty-postgres:5432/realty_parser?sslmode=disable"

logging:
  level: "info"   # debug / info / warn / error
  file_path: "/var/log/subscription-handler/app.log"

metrics:
  port: 9094
```

## Метрики

Доступны на порту `9094` (на хосте):

```bash
curl http://localhost:9094/metrics
curl http://localhost:9094/healthz
```

| Метрика | Тип | Описание |
|---|---|---|
| `subscription_handler_messages_received_total` | Counter | Всего сообщений от пользователей |
| `subscription_handler_subscriptions_created_total` | Counter | Успешно созданных подписок |
| `subscription_handler_commands_total{command}` | Counter | Команд по типу (`/subscript`, `/cancel`, ...) |

## Запуск в Docker

Сервис запускается в сети `realty-net` и требует только PostgreSQL.

```bash
# 1. PostgreSQL (если ещё не запущен)
cd /путь/к/realty-parser && bash psql_setup.sh

# 2. subscription-handler
cd /путь/к/subscription-handler && bash server_setup.sh
```

### Управление контейнером

```bash
# Логи
docker logs -f subscription-handler

# Остановить
docker stop subscription-handler
```

### Состояние диалога

Состояние сессии каждого пользователя хранится **в памяти** сервиса. При перезапуске контейнера незавершённые диалоги сбрасываются — пользователю потребуется начать `/subscript` заново.

### Порты в Docker-сети

| Порт на хосте | Порт в контейнере | Назначение |
|---|---|---|
| `9094` | `9094` | Метрики и healthz |
